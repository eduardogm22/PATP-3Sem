Introdução

    O controle de patrimônio é um processo de gerenciamento dos bens e ativos de uma empresa, o nosso objetivo central para o desenvolvimento se baseia em:

  • check_circleIdentificar, registrar e monitorar os bens
  • check_circleFornecer informações e relatórios financeiros

Planejamento do Design

O design foi planejado no semestre passado com interface feita em PyQt. Nesta fase, a implementação ocorreu com Scene Builder e JavaFX, mantendo a estrutura limpa e intuitiva. A cor verde foi escolhida por representar estabilidade, segurança e controle, essenciais para a gestão patrimonial.

Tecnologias Aplicadas

Sobre JavaFX/Scene Builder


JavaFX com Scene Builder é uma combinação usada para criar interfaces gráficas modernas em Java. O JavaFX fornece os recursos gráficos e de interação, enquanto o Scene Builder permite montar visualmente essas telas por meio de arrastar e soltar, gerando automaticamente os arquivos .fxml que definem a estrutura da interface. Essa abordagem facilita o desenvolvimento de aplicações com separação clara entre código e design, tornando o processo mais produtivo e organizado.

Sobre a API Python Java


O sistema foi desenvolvido com uma arquitetura bem definida, separando a lógica da apresentação. A API foi construída em Python, responsável por processar as requisições e executar as funções principais do sistema. Já a interface foi desenvolvida em Java, proporcionando uma estrutura visual robusta e organizada. Essa integração entre Python e Java oferece maior flexibilidade, facilita a manutenção e garante um funcionamento mais eficiente do sistema.


Sobre MySQL — Banco de Dadosc


MySQL é um sistema de gerenciamento de banco de dados relacional (SGBD) amplamente utilizado para armazenar, organizar e acessar dados de forma eficiente. Baseado na linguagem SQL (Structured Query Language), ele permite criar tabelas, relacionamentos e executar consultas de forma rápida e estruturada. É uma tecnologia robusta, confiável e gratuita, sendo usada em diversos sistemas, desde aplicações simples até plataformas web complexas.

Central de Ajuda - Sistema de Patrimônio

📂 Cadastros

Como cadastrar um novo bem?

Acesse o menu \"Bens\" > clique em \"Novo Bem\" > preencha os campos obrigatórios e clique em \"Salvar\".

Como editar dados de um patrimônio?

Localize o bem na lista, clique no ícone de editar, faça as alterações e clique em \"Atualizar\".

Como inativar ou excluir um bem?

Vá até o item desejado, clique em \"Mais ações\" e selecione \"Inativar\" ou \"Excluir\".

📂 Transferência de Bens

Como transferir um item de um setor para outro?

Acesse o menu \"Transferência\" > selecione o bem > escolha o setor de destino > clique em \"Confirmar\".

Registro de histórico de movimentação

Vá em \"Histórico\" no menu principal para visualizar todas as movimentações realizadas por item.

📂 Inventário

Como gerar inventário por setor?

Entre no menu \"Inventário\" > selecione o setor > clique em \"Gerar\" para visualizar os itens.

Como imprimir etiquetas?

No inventário, selecione os itens desejados e clique em \"Imprimir etiquetas\".

Como lidar com patrimônio não localizado?

Marque como \"não encontrado\" no inventário e acione a equipe responsável para investigação.

📂 Relatórios

Relatório de bens por setor

Menu \"Relatórios\" > selecione \"Bens por setor\" > escolha o setor > clique em \"Gerar relatório\".

Relatório de depreciação

Menu \"Relatórios\" > selecione \"Depreciação\" > configure o período > clique em \"Gerar\".

Exportação em Excel/PDF

Todos os relatórios possuem a opção de exportação via botão \"Exportar\" no canto superior direito.

📂 Usuários e Permissões

Como cadastrar novos usuários?

Menu \"Configurações\" > \"Usuários\" > clique em \"Novo usuário\" > preencha e salve.

Como restringir acessos por setor?

Ao cadastrar ou editar um usuário, defina os setores permitidos nas permissões de acesso.

📂 Erros comuns

“Não foi possível salvar” — causas e soluções

Verifique se todos os campos obrigatórios estão preenchidos e se há conexão com o servidor.

“Sem permissão para esta ação”

Solicite ao administrador do sistema para revisar seu perfil de permissões.

“Item não localizado no setor informado”

Verifique o histórico do bem. O item pode ter sido movido para outro setor ou inativado.

Quem somos nós?

Somos estudantes do curso de Análise e Desenvolvimento de Sistemas, atualmente no segundo ano da formação. Desenvolvemos um Sistema de Controle de Patrimônio com foco na gestão eficiente dos bens institucionais, utilizando Python para a lógica da aplicação, com uma API própria para integração e comunicação de dados. As interfaces gráficas foram construídas com JavaFX e Scene Builder, proporcionando uma navegação intuitiva e moderna. O banco de dados utilizado é o MySQL, garantindo confiabilidade no armazenamento das informações . O sistema foi projetado com exclusividade para atender às demandas da instituição IDEAU, reforçando nosso compromisso com soluções práticas, funcionais e aplicadas ao contexto real.
